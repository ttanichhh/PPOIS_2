# Лабораторная работа №4
## Разработка веб-приложения для системы «Персональный медицинский ассистент»

## Цель работы
Перевести существующую систему на один из фреймворков создания веб-приложений и реализовать полноценный веб-интерфейс с сохранением общей предметной логики.

## Используемый фреймворк
В работе использован фреймворк `Flask`.

## Постановка задачи
Необходимо было:

1. использовать веб-фреймворк для построения приложения
2. сохранить уже реализованную предметную область
3. реализовать веб-интерфейс для всех основных операций системы
4. обеспечить отображение текущего состояния модели
5. сохранить возможность повторного использования общего прикладного слоя

## Краткое описание системы
Приложение «Персональный медицинский ассистент» позволяет:

- регистрировать пациентов
- просматривать медицинскую историю
- записывать симптомы
- получать рекомендации
- добавлять лекарства и напоминания
- искать клиники по району
- получать консультации врачей
- сохранять данные в `JSON`

## Архитектура
Проект разделён на уровни:

- `entities/` содержит сущности предметной области
- `assistant/` содержит бизнес-логику и работу с хранилищем
- `app/application.py` содержит общий прикладной сервис
- `app/web.py` содержит веб-слой на `Flask`
- `templates/` и `static/` содержат HTML-шаблоны и стили

Такой подход позволил не дублировать бизнес-логику и использовать её через веб-интерфейс.

## Выделенный общий код для л/р №1 и л/р №4
Общий код, используемый и в консольном, и в веб-интерфейсе, сохранён в следующих модулях:

### Слой предметной области
- `entities/person.py`
- `entities/patient.py`
- `entities/doctor.py`
- `entities/medical.py`

В этих файлах находятся основные сущности системы:

- пациент
- врач
- симптом
- лекарство
- рекомендация
- медицинская история

### Слой бизнес-логики
- `assistant/medical_assistant.py`
- `assistant/storage.py`

Этот слой отвечает за:

- регистрацию пользователей
- запись симптомов
- выдачу рекомендаций
- добавление лекарств
- поиск клиник и врачей
- консультации врача
- загрузку и сохранение данных в `JSON`

### Общий прикладной слой
- `app/application.py`

Именно `app/application.py` выступает общей точкой доступа к операциям системы:

- регистрация пользователя
- получение списка пользователей
- просмотр истории
- запись симптомов
- получение рекомендаций
- добавление лекарств
- поиск клиник и врачей
- консультация врача
- сохранение данных

За счёт этого `CLI` и `web` не содержат предметную логику, а только вызывают общий прикладной слой.

## Интерфейсы, использующие общий код
### CLI
- `app/cli.py`

Консольный интерфейс не содержит собственной предметной логики. Он только вызывает методы `MedicalAssistantApp`.

### Web
- `app/web.py`
- `templates/index.html`
- `static/styles.css`

Веб-интерфейс на `Flask` также работает через `MedicalAssistantApp` и использует тот же общий слой, что и `CLI`.

## Схема взаимодействия
```text
CLI ----------\
               \
                > app/application.py -> assistant/medical_assistant.py -> entities/*
               /
Web ----------/
```

## Возможности веб-приложения
Веб-интерфейс реализует:

- регистрацию пользователей
- просмотр списка пользователей
- просмотр медицинской истории выбранного пациента

## Основные файлы
### `app/application.py`
Общий прикладной слой. Предоставляет единый API для интерфейсов.

### `app/web.py`
Содержит маршруты Flask, обработку форм, загрузку данных для страницы и сохранение после изменений.

### `templates/index.html`
Основной HTML-шаблон веб-приложения.

### `static/styles.css`
Оформление интерфейса.

### `assistant/storage.py`
Загрузка и сохранение данных в файле `JSON`.

## Запуск приложения
### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск веб-приложения
```bash
python3 main.py web
```

По умолчанию приложение будет доступно по адресу:

```text
http://127.0.0.1:5000
```

### Дополнительные параметры
```bash
python3 main.py web --host 0.0.0.0 --port 8000
```

## Поддерживаемые режимы
В проекте используются два режима запуска:

```bash
python3 main.py cli
python3 main.py web
```

## Демонстрация работоспособности CLI и web с общим кодом
Работоспособность демонстрируется за счёт использования одного и того же общего прикладного слоя `app/application.py` в двух режимах:

1. пользователь, созданный через `web`, отображается в `CLI`
2. симптом, записанный через общий прикладной слой, отображается в веб-интерфейсе

Проверить это можно вручную:

```bash
python3 main.py web
python3 main.py cli
```
