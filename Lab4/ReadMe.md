# Лабораторная работа №4
## Разработка графического интерфейса для системы «Персональный медицинский ассистент»

## Цель работы
Разработать графический пользовательский интерфейс для программной системы из л/р №1, сохранить общий код предметной области и обеспечить возможность запуска приложения как в консольном, так и в графическом режиме.

## Постановка задачи
В рамках лабораторной работы требовалось:

1. использовать архитектурный шаблон `MVC` или `MVP`
2. реализовать интерфейс, включающий все операции управления моделью
3. обеспечить отображение текущего состояния модели
4. сохранить общий код л/р №1 и л/р №4
5. предусмотреть запуск приложения как с `GUI`, так и с `CLI`

## Краткое описание системы
Приложение «Персональный медицинский ассистент» предназначено для работы с медицинской информацией пользователя. Система позволяет:

- регистрировать пользователей
- хранить медицинскую историю
- записывать симптомы
- выдавать рекомендации
- добавлять лекарства и напоминания
- искать клиники по району
- получать консультацию врача
- просматривать текущее состояние системы

В качестве хранилища данных используется файл `JSON`.

## Архитектурное решение
Для графического интерфейса использован шаблон `MVP`.

### Model
Слой модели содержит предметную логику и данные приложения:

- `entities/` — сущности предметной области
- `assistant/medical_assistant.py` — основная бизнес-логика
- `assistant/storage.py` — загрузка и сохранение данных

### Presenter
Посредник между интерфейсом и моделью:

- `app/gui.py`
- класс `MedicalAssistantPresenter`

Presenter принимает действия пользователя из GUI, вызывает операции прикладного слоя и передаёт результаты в представление.

### View
Графическое представление приложения:

- `app/gui.py`
- класс `MedicalAssistantGUI`

View отвечает за отображение вкладок, таблиц, форм ввода, списков и текстовых областей.

### Общий прикладной слой
Чтобы не дублировать код между CLI и GUI, общий набор операций вынесен в:

- `app/application.py`
- класс `MedicalAssistantApp`

Этот слой используется обоими интерфейсами и предоставляет единые методы для:

- регистрации пользователя
- получения списка пользователей
- просмотра истории
- работы с симптомами
- работы с лекарствами
- поиска клиник и врачей
- сохранения данных

## Назначение основных модулей
### `entities/person.py`
Содержит базовый класс `Person`, от которого наследуются пациент и врач.

### `entities/patient.py`
Описывает пользователя системы и хранит возраст и медицинскую историю.

### `entities/doctor.py`
Описывает врача: имя, телефон, специализацию и клинику.

### `entities/medical.py`
Содержит классы `Symptom`, `Medication`, `Recommendation`, `MedicalHistory`.

### `assistant/medical_assistant.py`
Реализует основные операции предметной области:

- регистрация пользователя
- запись симптома
- получение совета
- добавление лекарства
- поиск клиник
- выбор врача
- консультация

### `assistant/storage.py`
Отвечает за чтение и сохранение данных в `JSON`.

### `app/application.py`
Содержит общий прикладной слой, который объединяет модель и интерфейсы.

### `app/cli.py`
Реализует консольный интерфейс приложения.

### `app/gui.py`
Реализует графический интерфейс на базе `tkinter`.

### `main.py`
Содержит единую точку входа и позволяет выбрать режим запуска.

## Реализация пользовательского интерфейса
Графический интерфейс включает вкладки:

- `Пользователи`
- `Медицинские операции`
- `Клиники и врачи`
- `Состояние системы`

### Вкладка «Пользователи»
Позволяет:

- зарегистрировать нового пользователя
- увидеть список всех пользователей
- выбрать пользователя
- просмотреть его медицинскую историю

### Вкладка «Медицинские операции»
Позволяет:

- добавить симптом
- получить совет по симптому
- добавить лекарство и расписание
- просмотреть список лекарств выбранного пользователя

### Вкладка «Клиники и врачи»
Позволяет:

- выбрать район
- просмотреть клиники района
- просмотреть список врачей
- выбрать врача
- отправить вопрос врачу и получить консультацию

### Вкладка «Состояние системы»
Отображает:

- имя ассистента
- количество пользователей
- список врачей
- список районов
- текущее содержимое модели

## Реализация CLI
Консольный интерфейс сохранён как отдельный режим запуска. Он использует тот же общий прикладной слой, что и GUI, поэтому бизнес-логика не дублируется.

Таким образом, выполнено требование сохранения общего кода для л/р №1 и л/р №4.

## Поддерживаемые режимы запуска
### Запуск CLI
```bash
python3 main.py cli
```

### Запуск GUI
```bash
python3 main.py gui
```

### Справка по параметрам запуска
```bash
python3 main.py --help
```

## Пример сценария использования
1. Пользователь запускает приложение в режиме `gui`
2. На вкладке «Пользователи» регистрирует пациента
3. На вкладке «Медицинские операции» добавляет симптом
4. Получает рекомендацию по симптому
5. Добавляет лекарство и расписание приёма
6. На вкладке «Клиники и врачи» выбирает район
7. Просматривает клиники и врачей
8. Отправляет вопрос выбранному врачу
9. Сохраняет данные
