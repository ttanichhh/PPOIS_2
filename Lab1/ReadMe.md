# Лабораторная работа
## Персональный медицинский ассистент

## Описание проекта
Проект реализует CLI-систему персонального медицинского ассистента. Пользователь может регистрироваться, сохранять симптомы, получать рекомендации, хранить напоминания о лекарствах, искать клиники и консультироваться с врачами.

В усложненном варианте были добавлены:

- явная модель состояний системы;
- сохранение состояния между запусками;
- декомпозиция на доменные сущности, сервисный слой и слой хранения;
- UML-диаграмма классов и UML-диаграмма состояний;
- обновленные unit-тесты.

## Предметная область
Ключевые сущности:

- пользователь;
- ассистент;
- медицинская история;
- симптом;
- рекомендация;
- лекарство;
- врач;
- клиника.

Основные операции:

- регистрация пользователя;
- запись симптомов;
- предоставление медицинских советов;
- напоминание о приеме лекарств;
- поиск ближайшей клиники;
- консультация с врачом;
- просмотр медицинской истории;
- сохранение состояния системы.

## Архитектура
Проект разделен на несколько уровней:

- `entities/` - доменные сущности предметной области;
- `assistant/medical_assistant.py` - основной сервис приложения;
- `assistant/storage.py` - абстракция хранилища и JSON-реализация;
- `assistant/state.py` - модель состояний и правила переходов;
- `main.py` - CLI-интерфейс;
- `tests/` - unit-тесты.

Такое разделение помогает показать принципы SOLID:

- `S`: классы сущностей отвечают только за свои данные и валидацию;
- `O`: систему можно расширять новыми хранилищами без переписывания сервиса;
- `L`: `Patient` и `Doctor` корректно работают как наследники `Person`;
- `I`: интерфейс хранилища отделен от конкретной JSON-реализации;
- `D`: `MedicalAssistant` зависит от абстракции `StorageProtocol`, а не от жестко привязанного файла.

## Сохранение состояния
Состояние системы хранится в `data.json` в секции `system_state`.

Сохраняются:

- текущий статус системы;
- последнее выполненное действие;
- список пользователей;
- медицинская история пользователей.

После повторного запуска ассистент восстанавливает данные и переводится в рабочее состояние.
