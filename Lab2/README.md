# Система управления записями клиентов (Lab2)

Настольное приложение на **Python + PySide6** для работы с массивом записей клиентов: добавление, поиск, удаление, постраничный вывод, импорт/экспорт **XML (DOM/SAX)** и хранение в **SQLite**.

---

## Описание проекта

Приложение реализует требования учебного задания:

- одно главное окно + несколько диалогов (через меню; команды также дублируются на панели инструментов)
- построение по шаблону **Model–View–Controller**
- ввод записей через диалог
- поиск в отдельном диалоге (результаты выводятся в этом же диалоге)
- удаление в отдельном диалоге (с сообщением о количестве удалённых записей)
- отображение текущего массива записей в главном окне
- сохранение/загрузка массива в XML через стандартные диалоги
- экспорт в XML через **DOM**, импорт через **SAX**
- постраничный вывод записей (main window и search dialog)

---

## Архитектура MVC

### Model

Model отвечает за данные и бизнес-логику:

- `ClientRecord` — структура записи клиента
- `ClientRepository` — работа с базой **SQLite** (CRUD + поиск + удаление + пагинация)
- `SearchCriteria`, `CriteriaMode` — условия поиска/удаления
- `xml_export_dom.py` — экспорт в XML через DOM
- `xml_import_sax.py` — импорт из XML через SAX

### View

Представления (PySide6):

- `MainWindow` — главное окно (таблица + пагинация)
- `AddEditDialog` — диалог добавления записи
- `SearchDialog` — диалог поиска (условия + результаты + пагинация)
- `DeleteDialog` — диалог удаления (условия)
- `PaginationWidget` — элемент управления страницами
- `ClientsTableModel` — модель таблицы для `QTableView`

### Controller

Связывает Model и View:

- `MainController` — обрабатывает действия пользователя:
  - добавление записи
  - поиск (с постраничным выводом в диалоге поиска)
  - удаление (с сообщением о результате)
  - импорт/экспорт XML
  - постраничный вывод в главном окне

## Структура данных

Каждая запись клиента содержит поля:

| Поле | Описание |
|------|----------|
| ФИО | Полное ФИО клиента |
| Номер счета | Строка  |
| Адрес прописки | Строка |
| Мобильный телефон | Строка |
| Городской телефон | Строка |

Хранение: **SQLite** (`Lab2/clients.sqlite3`).

---

## Постраничный вывод

Пагинация реализована в:

- главном окне (список всех записей)
- диалоге поиска (результаты поиска)

Возможности:

- переход на первую/последнюю страницу
- переход на предыдущую/следующую страницу
- выбор количества записей на странице (например 5/10/20/50)
- отображение текущей страницы и общего количества страниц, а также общего числа записей

## Поиск

Поиск выполняется в **отдельном диалоговом окне**.

Поддерживаются 3 режима:

### 1) Мобильный ИЛИ фамилия

- мобильный: **полное совпадение по цифрам** (форматирование `+`, пробелы, `-` игнорируются)
- фамилия: сравнивается с **первым словом в ФИО** (например, `Иванов Иван Иванович` → фамилия `Иванов`)
- если заполнены оба поля — логическое **ИЛИ**

### 2) Счёт ИЛИ адрес (по части)

- номер счёта: поиск по **части строки**
- адрес: поиск по **части строки** (без учёта регистра)
- логическое **ИЛИ**

### 3) ФИО (часть) И телефон (полностью)

- ФИО: поиск по **части строки** (без учёта регистра)
- телефон: **полное совпадение по цифрам**, проверяется **мобильный или городской**
- логическое **И**

---

## Удаление

Удаление записей выполняется в **отдельном диалоговом окне**.

Правила и режимы совпадают с режимами поиска.  
После удаления приложение показывает сообщение:

- сколько записей удалено, **или**
- что записей по условиям не найдено

---

## Импорт и экспорт XML

### Экспорт (DOM)

Сохранение выполняется через стандартный диалог “Save File”.

- Экспортируются **все записи** из базы
- Используется DOM (`xml.dom.minidom`)
- Функция: `export_clients_to_xml_dom(records, file_path)`

### Импорт (SAX)

Загрузка выполняется через стандартный диалог “Open File”.

- Записи из XML **добавляются** в базу данных
- Используется SAX (`xml.sax`)
- Функция: `import_clients_from_xml_sax(file_path)`

## Сценарии демонстрации

### Сценарий 1 — запуск

1. Запустить приложение.
2. Открывается главное окно.
3. Отображается таблица клиентов + пагинация.

### Сценарий 2 — добавление записи

1. **Записи → Добавить запись**
2. Ввести ФИО, счёт, адрес, телефон(ы)
3. Нажать **Сохранить**
4. Запись появляется в таблице

### Сценарий 3 — поиск

1. **Записи → Поиск**
2. Выбрать режим поиска
3. Ввести условия
4. Нажать **Найти**
5. Результаты отображаются в таблице + работает пагинация

### Сценарий 4 — удаление

1. **Записи → Удаление**
2. Выбрать режим
3. Ввести условия
4. Нажать **Удалить**
5. Появляется сообщение о количестве удалённых записей

### Сценарий 5 — экспорт XML

1. **Файл → Сохранить XML**
2. Выбрать имя файла
3. Получить XML с текущими записями

### Сценарий 6 — импорт XML

1. **Файл → Загрузить XML**
2. Выбрать XML файл
3. Записи добавляются в базу, отображаются в главном окне
