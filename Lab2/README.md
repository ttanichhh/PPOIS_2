# Система учёта клиентов

Настольное приложение на `Python + PySide6`, построенное по шаблону `Model-View-Controller`.

Программа поддерживает:

- ввод записей через отдельный диалог;
- поиск в отдельном диалоге с выводом результата в этом же окне;
- удаление по условиям в отдельном диалоге;
- постраничное отображение записей;
- альтернативный просмотр в виде таблицы и дерева;
- хранение данных в `SQLite`;
- загрузку и сохранение данных в `XML` и в отдельные файлы базы данных;
- экспорт в XML через `DOM` и импорт из XML через `SAX`.

## Структура проекта

- `app/model` — сущности, репозиторий SQLite, XML DOM/SAX, критерии поиска.
- `app/view` — главное окно, диалоги, таблица, пагинация.
- `app/controller` — контроллер, связывающий интерфейс и модель.
- `tools/generate_demo_xml.py` — генерация демонстрационных XML и SQLite файлов.
- `REPORT.md` — готовый отчёт по системе и сценарии демонстрации.

## MVC

### Model

- `ClientRecord` — модель записи клиента.
- `ClientRepository` — работа с SQLite: добавление, поиск, удаление, пагинация, переключение БД.
- `SearchCriteria`, `CriteriaMode` — описание условий поиска и удаления.
- `xml_export_dom.py` — экспорт XML через DOM.
- `xml_import_sax.py` — импорт XML через SAX.

### View

- `MainWindow` — главное окно.
- `AddEditDialog` — ввод записи.
- `SearchDialog` — поиск и вывод результатов.
- `DeleteDialog` — удаление по условиям.
- `PaginationWidget` — навигация по страницам.
- `ClientsTableModel` — модель таблицы.

### Controller

- `MainController` — обработка действий пользователя, загрузка страниц, импорт/экспорт XML и SQLite, переключение таблицы и дерева.

## Поля записи

- `fio` — ФИО клиента.
- `account_number` — номер счёта.
- `registration_address` — адрес прописки.
- `mobile_phone` — мобильный телефон.
- `home_phone` — городской телефон.

Все поля хранятся в подходящем типе `TEXT`, потому что в варианте задания нет числовых или календарных атрибутов. Номер счёта и телефоны сохраняются как строки, чтобы не терять ведущие нули, `+` и формат записи.

## Поиск и удаление

Реализованы три режима, соответствующие варианту 4:

1. По номеру телефона или фамилии.
2. По номеру счёта или адресу.
3. По ФИО и цифрам, встречающимся в одном из номеров.

Для поиска и удаления используются разные диалоги.

## Пагинация

Пагинация работает в главном окне и в окне поиска.

Показываются:

- текущая страница;
- общее число страниц;
- количество записей на текущей странице;
- общее количество записей;
- выбранный размер страницы.

## Хранение данных

### XML

- сохранение: стандартный диалог `Save File`;
- загрузка: стандартный диалог `Open File`;
- запись выполняется через `xml.dom.minidom`;
- чтение выполняется через `xml.sax`;
- при загрузке XML текущий набор записей заменяется содержимым файла.

### База данных

- текущее рабочее хранилище — `SQLite`;
- можно открыть любой файл БД через стандартный диалог;
- можно сохранить текущий набор записей в отдельный файл БД через стандартный диалог.
