# Отчёт по разработанной системе

## 1. Назначение системы

Разработана настольная информационная система для хранения и обработки массива записей клиентов.

Вариант 4 содержит поля:

- ФИО клиента;
- номер счёта;
- адрес прописки;
- мобильный телефон;
- городской телефон.

Система позволяет вводить, отображать, искать, удалять, сохранять и загружать записи как из XML-файлов, так и из базы данных SQLite.

## 2. Архитектура

Приложение построено по шаблону `Model-View-Controller`.

### Model

Слой модели содержит:

- `ClientRecord` — объект записи клиента;
- `ClientRepository` — доступ к данным в SQLite;
- `SearchCriteria` и `CriteriaMode` — описание вариантов поиска и удаления;
- `xml_export_dom.py` — экспорт данных в XML методом DOM;
- `xml_import_sax.py` — импорт данных из XML методом SAX.

### View

Слой представления содержит:

- `MainWindow` — главное окно приложения;
- `AddEditDialog` — окно ввода новой записи;
- `SearchDialog` — окно поиска с таблицей результатов;
- `DeleteDialog` — окно удаления по условиям;
- `PaginationWidget` — общий элемент постраничной навигации;
- `ClientsTableModel` — модель таблицы для отображения записей.

### Controller

Слой контроллера содержит:

- `MainController`, который обрабатывает действия пользователя и связывает интерфейс с моделью.

## 3. Реализованные функции

В системе реализованы следующие возможности:

1. Формирование массива записей через диалог добавления.
2. Поиск записей в отдельном диалоговом окне.
3. Вывод результатов поиска в том же диалоге, где задавались условия.
4. Удаление записей в отдельном диалоговом окне.
5. Сообщение пользователю о количестве удалённых записей или об отсутствии совпадений.
6. Отображение текущего массива записей в главном окне.
7. Постраничный вывод записей в главном окне и в окне поиска.
8. Альтернативный просмотр записей в виде дерева.
9. Хранение данных в базе `SQLite`.
10. Загрузка и сохранение данных в XML.
11. Открытие произвольного файла базы данных SQLite и сохранение текущего набора записей в новый файл базы данных.

## 4. Реализация условий варианта 4

Поиск и удаление выполняются в трёх режимах:

1. По номеру телефона или фамилии.
2. По номеру счёта или адресу.
3. По ФИО и цифрам, встречающимся в одном из телефонов.

Для первого режима телефон сравнивается по цифрам, чтобы формат записи не влиял на результат.

Для второго режима используются частичные совпадения по номеру счёта и адресу.

Для третьего режима выполняется поиск по части ФИО и точному совпадению телефонного номера среди мобильного или городского.

## 5. Организация хранения

### 5.1 База данных

Основное хранилище системы — `SQLite`.

Таблица `clients` содержит поля:

- `id` — идентификатор записи;
- `fio`;
- `account_number`;
- `registration_address`;
- `mobile_phone`;
- `home_phone`.

### 5.2 XML

Для записи XML используется `DOM`, для чтения — `SAX`, что соответствует требованиям задания.

Корневой элемент документа — `clients`, каждая запись хранится в элементе `client`.

## 6. Интерфейс пользователя

Главное окно включает:

- меню;
- панель инструментов;
- переключение режимов отображения `Таблица` / `Дерево`;
- область отображения записей;
- панель пагинации;
- строку состояния с путём к текущей базе данных.

В дереве каждая запись представлена отдельным узлом, а листовые элементы соответствуют конкретным полям записи.

## 7. Пагинация

Пагинация поддерживает:

- переход на первую страницу;
- переход на предыдущую страницу;
- переход на следующую страницу;
- переход на последнюю страницу;
- изменение числа записей на странице;
- показ количества записей на текущей странице;
- показ общего количества записей;
- показ номера текущей страницы и общего числа страниц.

## 8. Демонстрационные данные

Для демонстрации подготовлен скрипт `tools/generate_demo_xml.py`.

Он создаёт:

- 3 XML-файла;
- 3 SQLite-файла.

Каждый файл содержит не менее 60 уникальных осмысленных записей.

## 9. Сценарии демонстрации системы

### Сценарий 1. Запуск приложения

1. Запустить `python main.py`.
2. Показать главное окно.
3. Обратить внимание на таблицу, дерево, пагинацию и строку состояния.

### Сценарий 2. Добавление записи

1. Выбрать команду `Записи -> Добавить запись`.
2. Ввести данные клиента.
3. Сохранить запись.
4. Показать, что запись появилась в таблице и в дереве.

### Сценарий 3. Поиск записей

1. Открыть `Записи -> Поиск`.
2. Выбрать один из трёх режимов.
3. Ввести условия поиска.
4. Нажать `Найти`.
5. Показать найденные записи и работу пагинации внутри окна поиска.

### Сценарий 4. Удаление записей

1. Открыть `Записи -> Удаление`.
2. Выбрать режим удаления.
3. Ввести условия.
4. Нажать `Удалить`.
5. Показать сообщение о количестве удалённых записей.

### Сценарий 5. Работа с XML

1. Загрузить один из файлов `demo_xml`.
2. Показать заполнение таблицы.
3. Сохранить текущий массив записей в новый XML-файл.
4. Указать, что запись произведена через DOM, а чтение через SAX.

### Сценарий 6. Работа с базой данных

1. Открыть один из файлов `demo_db`.
2. Показать, что приложение переключилось на выбранную базу.
3. Добавить или удалить записи.
4. Выполнить `Сохранить БД как...` и сохранить текущий набор в новый файл SQLite.

### Сценарий 7. Альтернативное отображение

1. Переключиться из таблицы в дерево.
2. Раскрыть одну запись.
3. Показать, что листья дерева соответствуют отдельным полям записи.

## 10. Что было исправлено и как

По сравнению с более простым вариантом были внесены следующие доработки:

1. Добавлено полноценное хранение записей в `SQLite`, а не только работа с массивом в памяти.
2. Реализовано открытие произвольной базы данных через стандартный диалог.
3. Реализовано сохранение текущего набора записей в отдельный файл базы данных.
4. Добавлено альтернативное представление записей в виде дерева.
5. Улучшена пагинация: теперь показываются не только номера страниц, но и количество записей на текущей странице и всего.
6. Сообщение после удаления стало явно показывать число удалённых записей.
7. Генератор демонстрационных данных расширен: теперь он создаёт и XML, и SQLite-файлы.
8. Обновлена документация по системе и подготовлены готовые сценарии для демонстрации.

## 11. Вывод

В результате разработано приложение, удовлетворяющее требованиям задания для варианта 4. Система построена по MVC, поддерживает отдельные диалоги для добавления, поиска и удаления, использует SQLite и XML, а также предоставляет постраничное и древовидное отображение записей.
